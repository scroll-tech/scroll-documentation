یک بلوک مجموعه‌ای از معاملات است و به بلوک قبلی که به آن افزوده شده است، لینک می‌شود.

## هدر بلوک

هدر بلوک در Scroll ساختار مشابهی با اتریوم دارد. با این حال، برخی از فیلدهای هدر بلوک برای تطبیق با زمینه لایه 2 تغییر یافته‌اند، که در جدول زیر به تفصیل آمده است:

| فیلد               | توضیحات                                                             |
| ------------------- | ------------------------------------------------------------------ |
| `parentHash`        | هش بلوک والد.                                                       |
| `uncleHash`         | همیشه `keccak256(rlp([]))`.                                        |
| `coinbase`          | هزینه‌ها به آدرس قرارداد پیش‌توسعه یافته جمع‌آوری می‌شود: `0x5300000000000000000000000000000000000005`. در بیشتر موارد، فیلد `"miner"` در بازگشت RPC آدرس صفر `0x0000000000000000000000000000000000000000` است. |
| `stateRoot`         | هش ریشه درخت وضعیت. این هش Poseidon است.                           |
| `transactionsRoot`  | هش ریشه درخت معاملات.                                               |
| `receiptsRoot`      | هش ریشه درخت رسیدهای معاملات.                                       |
| `logsBloom`         | ساختار داده‌ای که شامل لاگ‌های رویداد است.                          |
| `difficulty`        | همیشه 1 یا 2.                                                        |
| `number`            | شماره بلوک.                                                          |
| `gasLimit`          | حداکثر گس مجاز در بلوک.                                             |
| `gasUsed`           | گس استفاده شده در بلوک.                                            |
| `timestamp`         | زمان بلوک.                                                           |
| `extraData`         | امضا توسط امضاکننده بلوک، به همراه داده‌های اضافی دلخواه.            |
| `mixHash`           | همیشه 0.                                                             |
| `nonce`             | همیشه 0.                                                             |
| `baseFee`           | در حال حاضر در Scroll خالی است زیرا EIP-1559 فعال نشده است.          |

## زمان بلوک

زمان بلوک به فاصله بین دو بلوک متوالی اشاره دارد. در Scroll، زمان بلوک بر روی ۳ ثانیه تنظیم شده است. کاهش زمان بلوک تجربه کاربری را بهبود می‌بخشد و مقیاس‌پذیری زنجیره Scroll را افزایش می‌دهد.

زمان بلوک ۳ ثانیه به صورت تلاش بهترین حالت حفظ می‌شود و توسط پروتکل تضمین یا اجرا نمی‌شود.
